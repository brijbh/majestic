# Transit Model

Branches map to lines: main, feature, development, release, hotfix, and other.

Pull requests map to trains. Open pull requests move through review and checks. Merged pull requests arrive at the merge station. Releases and deployments become terminal trains when data is available.

Commits become activity events rather than separate trains. Review and check state changes influence train status: approved, moving, blocked, failed, merged, released, or deployed.

Stations are stable: Start, Development, Pull Request, Review, Checks, Merge, Release, and Deploy.
